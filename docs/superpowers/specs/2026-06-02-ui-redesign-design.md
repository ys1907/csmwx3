# 设计：盲盒首页重构 · UI/UX 重设计

- 日期：2026-06-02
- 状态：已批准（用户「定稿」）
- 主题：把首页从「一进来就显示推荐 + 换一个」改造为「盲盒点击揭晓」；统一视觉基调；精简功能；为揭晓设计三档稀有度动画。
- 视觉参考：`.superpowers/brainstorm/ui-redesign/`（01~13 可交互 mockup，本文件为权威文字描述，mockup 为补充）。

## 背景与目标

当前 app 玩法过多（今日推荐/转盘/塔罗/默契PK/盲盒/凑一桌/场景），首页一进来直接给结果，缺少悬念；动画不足；视觉只到「简洁」。

目标：**专注「帮两个人快速决定吃什么」**，首页是一个干净的盲盒——点击才揭晓，揭晓有「完美到极致」的分档动画；整体温馨、整洁、好看。

约束（不变）：原生微信小程序 WXML/WXSS/JS，无后端、无第三方依赖，纯逻辑与 `wx` 运行时隔离、可单测。

---

## 1. 视觉基调 ——「奶油恋人」（A+B 融合）

奶油暖底（温馨食欲）+ 马卡龙柔粉/薄荷点缀（情侣甜美）。

| 角色 | 色值 | 用途 |
|---|---|---|
| 奶油底 | `#FCF7F1` | 页面背景 |
| 卡片 | `#FFFFFF` | 盒子卡 / 菜卡 |
| 暖橙 / 深 | `#FF9466` / `#FF7A45` | **行动与食物**：主按钮、CTA、强调 |
| 柔粉 / 软 | `#EBA9C4` / `#FBEAF1` | **收藏与情侣**：♥、情侣元素 |
| 薄荷 / 软 | `#9FD8C8` / `#E6F4EF` | **清新标签**：场景/时间 pill |
| 暖棕 / 次 | `#4A3F3A` / `#9B8D85` | 文字 / 次要文字 |
| 分隔线 | `#EFE7DD` | hairline |
| 金（SSR） | `#FFC36B` / `#F5A623` / `#FFE7BE` | 仅稀有揭晓使用 |

**整洁原则**：三个彩色各司其职、不混用（暖橙只做行动、柔粉只做收藏/情侣、薄荷只做标签）。这是「整洁」的来源。落地时改写 `app.wxss` 的 design token（保留变量名，替换值），旧页面引用自动生效。暗黑模式后续单独处理（本期可暂保留现有暗黑 token 或简单适配）。

## 2. 信息架构

- 底部 **两个 tab**：**推荐**（盲盒首页）、**我的**。
- **凑一桌**：首页的低调副入口（不占 tab）。
- 筛选（场景/预算/口味/避免/排除近期）收进首页右上角 **⚙**，默认全场景。

---

## 3. 首页 · 盲盒态（默认）

- 顶部：标题「今天吃什么」+ 日期/餐段（如「6月2日 周一 · 晚餐」）+ 右上 ⚙（筛选）。**无场景 chip 常驻**。
- 中央：**盒子卡**（全屏唯一焦点）——白卡、大圆角、🎁、文案「轻触揭晓今天的命运」、三个跳动小点暗示可点；卡后有暖橙→柔粉的呼吸光晕。
- 盒子下方：低调的「🍱 两个人？帮你们凑一桌」副入口。
- 交互：**点盒子即揭晓**（无独立「开启」按钮——已确认删除）。

## 4. 首页 · 揭晓流程与三档动画（核心）

点击盒子触发揭晓，分两步独立计算：
1. **掷稀有度**（纯概率，与菜无关）：`rollRarity(rng)` → R 80% / SR 15% / SSR 5%。
2. **选菜**：用现有 `weightedPick` 在当前筛选池里选一道（沿用偏好/季节加权等逻辑）。

**关键不变量：点击前盒子三档完全一致，绝不泄露稀有度**；稀有度只决定揭晓动画的隆重度，**与抽到什么菜无关**（蛋炒饭也可能 SSR 降临）。废弃旧盲盒「按 budget 映射 R/SR/SSR」的逻辑。

三档动画（揭晓时按掷出的档位播放）：

- **R · 温柔升起**（80%）：盒子化作柔光上浮淡出，菜卡从原位 spring 升起淡入。朴素、不喧哗——保证日常的简洁。
- **SR · 凝光降临**（15%）：盒子蓄力轻抖 → 金光闪爆（暖金径向）→ 菜卡 popIn 弹出 + emoji 跳动 + 星粒四散；菜卡带 2px 金描边。
- **SSR · 殿堂降临**（5%，B++）：**暖色梦幻光氛**（暖金→柔粉→淡紫径向，提亮聚焦，**非黑场**）渐显 → 更久的蓄力 → 光柱升起 + 双金光环扩散 + 金爆 → 菜卡 popIn + **全息彩虹镀边**（conic 渐变旋转）+ 👑皇冠落下 + 星粒漫天 + 流光循环扫过。封神时刻。

平时多为 R（简洁），SSR 是稀客——「极致动画」不喧宾夺主。

## 5. 首页 · 揭晓态布局

揭晓后页面分为**两个独立部分**（不塞进同一张卡、无卡内分隔线）：

- **① 菜卡（内容）**：白卡，与盲盒态的盒子卡是「同一位置、同一种卡的两个状态」（容器→内容，本质是两个东西，由开盒动画衔接）。含：♥收藏（右上角圆钮，柔粉）、大 emoji、菜名、一句温馨理由、**小巧标签**（时间 warm pill / 场景 mint pill，font ~11px）。纯内容，无按钮。
- **② 操作（卡外，下方）**：「就吃这个」**居中精致胶囊**主按钮（暖橙渐变，height ~48、padding 横向，非全宽大块）+ 「↻ 换一个」小文字按钮（次要、灰）。

交互语义：
- **就吃这个**：确认决定，记入历史（沿用 `addToHistory`）。
- **换一个**：盒子快速重开、**再抽一次**（重新掷稀有度 + 重新选菜），保持盲盒手感。

## 6. 凑一桌

首页副入口进入；揭晓「一桌多道菜」（沿用 `foodLogic.buildMealCombo`，品类尽量不重复），可「换一桌」。视觉套用奶油恋人。

## 7. 功能精简

| 功能 | 处置 | 说明 |
|---|---|---|
| 今日推荐 | **改造为盲盒首页核心** | 进来是盒子，点击揭晓 |
| 转盘 | **移除（并入盲盒）** | 随机+悬念由盲盒揭晓动画统一承载 |
| 旧盲盒玩法 | **并入首页主交互** | 稀有度改纯概率 |
| 塔罗食运 | **移除** | gimmick、关联弱、运势 filter 引旧分类 |
| 默契 PK | **移除** | 不直接产出「吃什么」，偏离核心 |
| 凑一桌 | **保留** | 首页副入口 |
| 场景（外卖/堂食/自己做/聚餐） | **降为筛选** | 收进 ⚙ |
| 收藏 / 历史 / 口味画像 / 决策连胜 / 菜品管理 / 设置 | **收进「我的」** | 不占首页 |

## 8.「我的」页

收藏列表、历史记录、口味画像（`buildTasteProfile`）、决策连胜（`computeStreak`，轻量小激励）、菜品管理入口（跳 `pages/manage`）、设置（暗黑模式跟随、数据导入/导出）。

## 9. 技术架构

**纯逻辑（`utils/foodLogic.js`，可单测、rng 可注入）**
- 新增 `rollRarity(rng)` → 返回 `'R' | 'SR' | 'SSR'`，按累积概率 0.80/0.15/0.05 划分；rng 可注入。
- 复用：`filterFoods` / `weightedPick`（选菜）、`buildMealCombo`（凑一桌）、`buildTasteProfile` / `computeStreak`（我的页）、`inferSeason`（季节加权，已有）。

**页面（`pages/index/`）**
- 重构 `index.wxml` / `index.wxss` / `index.js`：首页 = 盲盒态 ↔ 揭晓态两状态机；移除转盘/塔罗/PK 的 WXML/WXSS/JS（index.js 大幅瘦身）。
- 揭晓动画用 **WXSS keyframes + class 切换**驱动（参考现有 wheel 的「CSS 动画为主、setData 控状态」模式，避免逐帧 setData）；三档对应三组动画 class。
- 大数组（`_foods` 等）仍存实例属性，不进 `data`。
- 「我的」：可作为 index 内的 tab 视图切换，或新增 `pages/mine/`。倾向 index 内 tab 切换以复用 `_foods`/storage，减少跨页重建（实现阶段定）。

**复用与清理**
- 复用：`storage.safeGet/safeSet`、`migrateFood`、`data/options.js` 常量、`data/sounds.js`（揭晓音效，可分档）。
- 清理：`pkData` 相关（PK 移除后不再写入；storage 旧键可保留不动，不主动迁移）。
- design token 改写集中在 `app.wxss`。

## 10. 测试策略

- `rollRarity`：注入确定性/序列 rng，验证三档边界（0/0.79/0.80/0.94/0.95/0.99）映射正确；大样本分布近似 80/15/5。
- 揭晓「稀有度与选菜解耦」：验证 `rollRarity` 与 `weightedPick` 互不影响（同一菜可得任意档）。
- 复用逻辑（filterFoods/weightedPick/buildMealCombo 等）已有测试，保持绿。
- 页面动画/交互无法单测；维持现有 `pageLifecycle.test.js` 风格覆盖生命周期，不为动画强造测试。

## 11. 实现阶段建议（供 plan 拆分）

1. **视觉 token**：改写 `app.wxss` 为奶油恋人配色（角色分工），验证两页基本变色。
2. **纯逻辑**：`foodLogic.rollRarity` + 测试（TDD）。
3. **盲盒首页骨架**：index 盲盒态 ↔ 揭晓态状态机 + 菜卡/操作分离布局（静态，无华丽动画）。
4. **三档揭晓动画**：R/SR/SSR 的 WXSS keyframes，接 `rollRarity`。
5. **功能移除**：删转盘/塔罗/PK 代码与 WXML/WXSS。
6. **筛选收纳**：场景/预算等收进 ⚙ sheet。
7. **「我的」页**：收藏/历史/画像/连胜/管理入口/设置。
8. **凑一桌**：套新视觉，接副入口。

视觉/动画实现用 **ui-ux-pro-max** 指导（设计原则落到 WXSS，其组件代码不可直接用——栈是 WXML/WXSS）；逻辑用 **TDD**。

## 12. 超出范围（YAGNI）

- 不做后端、不联网；稀有度纯前端随机。
- 不接真实定位/天气（季节加权已是纯本地）。
- 暗黑模式精细适配可后续单独处理。
- 不动既有 storage 旧键的迁移（pkData 等留存不清理）。

## 13. 未决细节（实现阶段确认，非阻塞）

- 「我的」用 index 内 tab 还是独立页 `pages/mine/`（倾向前者）。
- 揭晓音效是否分档（R 轻「叮」/ SSR 华丽音效），需 `data/sounds.js` 支持。
- ⚙ 筛选 sheet 的具体字段排布（沿用现有 filters 维度）。
